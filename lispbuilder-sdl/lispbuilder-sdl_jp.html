［スペシャル変数］ *default-color*

キーワードパラメータで COLOR を受け入れる関数は COLOR が指定されない場合に、デフォルトでシンボル *DEFAULT-COLOR* に束縛する可能性が高い。

COLOR は後述のマクロ WITH-COLOR により、 *DEFAULT-COLOR* に束縛される。

例
(DRAW-BOX A-BOX :SURFACE SDL:*DEFAULT-DISPLAY* :COLOR SDL:*BLACK*)
(DRAW-BOX B-BOX :SURFACE SDL:*DEFAULT-DISPLAY* :COLOR SDL:*BLACK*)
(DRAW-BOX C-BOX :SURFACE SDL:*DEFAULT-DISPLAY* :COLOR SDL:*BLACK*)

上記は、 *DEFAULT-COLOR* を *BLACK* へ設定することにより、短縮化できる。 

(WITH-SURFACE (DISP SDL:*DEFAULT-DISPLAY*)
  (WITH-COLOR (COL SDL:*BLACK*)
    (DRAW-BOX A-BOX)
    (DRAW-BOX B-BOX)
    (DRAW-BOX C-BOX)))

［スペシャル変数］ *default-display*

シンボル *DEFAULT-DISPLAY* は、関数 WINDOW により、その時点の表示面 DISPLAY-SURFACE に束縛される。

［スペシャル変数］ *default-font*

キーワードパラメータで FONT を受け入れる関数は FONT が指定されない場合に、デフォルトでシンボル *DEFAULT-FONT* に束縛する可能性が高い。

マクロ WITH-DEFAULT-FONT および WITH-FONT 、 INITIALISE-DEFAULT-FONT により、FONT は *DEFAULT-FONT* に束縛される。

例
(draw-string-solid-* "draw string centered" 100 100
                      :justify :center :color sdl:*white* :font a-font)
(draw-string-solid-* "draw string left" 100 100
                      :justify :left :color sdl:*white* :font a-font)
(draw-string-solid-* "draw string right" 100 100
                      :justify :right :color sdl:*white* :font a-font)

上記は、 *DEFAULT-FONT* を a-font へ設定することにより、短縮化できる。 

(WITH-DEFAULT-FONT (a-font)
  (WITH-COLOR (COL SDL:*WHITE*)
    (DRAW-STRING-SOLID-* "draw string centered" 100 100 :JUSTIFY :CENTER)
    (DRAW-STRING-SOLID-* "draw string left" 100 100 :JUSTIFY :LEFT)
    (DRAW-STRING-SOLID-* "draw string right" 100 100 :JUSTIFY :RIGHT)))

［スペシャル変数］ *default-surface*

キーワードパラメータで SURFACE を受け入れる関数は、 SURFACE が指定されない場合に、デフォルトでシンボル *DEFAULT-SURFACE* に束縛する可能性が高い。

次のマクロにより、SURFACE は *DEFAULT-SURFACE* に束縛される。： マクロ WITH-SURFACE およびマクロ WITH-SURFACES 。

例
(DRAW-SURFACE SURF-1 :SURFACE SDL:*DEFAULT-DISPLAY*)
(DRAW-SURFACE SURF-2 :SURFACE SDL:*DEFAULT-DISPLAY*)
(DRAW-SURFACE SURF-2 :SURFACE SDL:*DEFAULT-DISPLAY*)

上記は、*DEFAULT-SURFACE* を表示面へ設定することにより、短縮化できる。 

(WITH-SURFACE (DISP SDL:*DEFAULT-DISPLAY*)
  (DRAW-SURFACE SURF-1)
  (DRAW-SURFACE SURF-2)
  (DRAW-SURFACE SURF-2))

※キーワードパラメータ SURFACE のデフォルト値が *DEFAULT-DISPLAY* であるので、マクロ with-surface に *DEFAULT-DISPLAY* を指定する上記例は無意味。

［関数］ blit-surface source &optional surface => result

SOURCE サーフェスの、デスティネーション SURFACE への高速ブリットを実行する。 SOURCE セルにより定義される領域は、デスティネーションクリッピング矩形により定義される領域へブリットされる。ブリット関数は、ロックされたサーフェス上では呼び出されるべきではない。ブリット操作の結果は、ソースのサーフェスにサーフェスアルファチャネルが設定されているかにより、大きく異なる。カラーキーとアルファ属性が、サーフェスと相互作用する方法の優先順位は、次の通りである。：

  • ALPHA ＋ PIXEL-ALPHA を持つソースサーフェス ： ピクセルごとのアルファのみ使用するブリット。
  • ALPHA ＋ COLOR-KEY を持つソースサーフェス ： カラーキーおよびサーフェスごとのアルファ値を使用するブリット。
  • ALPHA を持つソースサーフェス ： カラーキーおよびサーフェスごとのアルファ値を使用するブリット。
  • COLOR-KEY を持つソースサーフェス ： カラーキーを使用したブリット。
  • ソースサーフェス ： 不透明な矩形ブリットを使用するブリット。

アルファチャネルは、ブリット操作に次の影響を及ぼす。：

RGBA から RGB
  • アクセサ ALPHA-ENABLED-P を伴う RGBA から RGB ： ソースは、アルファチャネルを使用して、デスティネーションとアルファブレンドされる。アクセサ COLOR-KEY-ENABLED-P およびサーフェスごとのアルファは無視される。
  • アクセサ ALPHA-ENABLED-P を伴わない RGBA から RGB ： RGB データはソースからコピーされる。ソースのアルファチャネルおよびサーフェスごとのアルファ値は無視される。アクセサ COLOR-KEY-ENABLED-P が設定されている場合、カラーキー値と一致しないピクセルのみがコピーされる。
RGB から RGBA

Draw a bezier curve of COLOR to SURFACE. The shape of the Bezier curve is defined by several control points. A control point is a vertex containing an X and Y coordinate pair.

パラメータ
  • :VERTICES is a list of control points of POINT.
  • :STYLE describes the line style used to draw the curve and may be one of :SOLID, :DASH, or :POINTS. Use :SOLID to draw a single continuous line through the specified waypoints. Use :DASH to draw a line between alternate waypoint pairs. Use :POINTS to draw a single pixel at each waypoint.

  • :SURFACE is the target SURFACE.

SOURCE の画像から新しい SURFACE を生成して返す。あるいは、 SOURCE が有効な画像を含まない場合、あるいは画像種別が判別できない場合、 NIL を返す。

マジックナンバーが存在する場合、画像種別の判別に使用される。マジックナンバーが利用できない（ TGA のような画像フォーマットは、マジックナンバーを含まない。）時に画像をロードするには、 :IMAGE-TYPE を使用して画像種別を指定する。 TGA のような、全てのマジック不可の画像フォーマットは、 IMAGE-TYPE を使用して指定される必要がある。

パラメータ
  • SOURCE は画像である。
  • COLOR-KEY はカラーキーを設定し、カラーキーをオンにする。
  • 'COLOR-KEY-AT' は、 POINT x/y にあるピクセルの色をカラーキーとして使用し、カラーキーをオンにする。
  • ALPHA はサーフェスの透明度を設定し、アルファブレンディングをオンにする。 0 ～ 255 の範囲である必要があり、 255 は不透明であり、 0 は透明である。

    (LOAD-IMAGE "image.bmp")

SDL_SetVideoMode を使用して、ピクセル幅 WIDTH および高さ HEIGHT の新しい SDL ウィンドウを生成する。

ピクセルごとの操作、あるいはアルファチャネルを伴うブリットサーフェスの実行を行うつもりで、かつ高いフレームレートが必要な場合は、（ FLAGS に） SDL-SW-SURFACE を使用する。 SDL-HW-SURFACE のようなハードウェアサーフェスを使用する時、ビデオメモリとシステムメモリをロックする時に、 SDL はサーフェスをビデオメモリからシステムメモリへコピーし、そしてそれらをロック解除する時にコピーし戻す。これは大きなパフォーマンス低下を引き起こす。（ハードウェアサーフェスを要求しても、ソフトウェアサーフェスを受け取るかも知れない点に注意。多くのプラットフォームは、 SDL-FULL-SCREEN を使用する時にのみ、ハードウェアサーフェスを提供できる。）　ブリットを行うサーフェスがビデオメモリにも配置できる時、 SDL-HW-SURFACE が最適である。

注釈： ウィンドウ化されたサーフェスの生成時、画面上の位置を制御するには、環境変数 SDL_VIDEO_CENTERED=center あるいは SDL_VIDEO_WINDOW_POS=x,y を設定する。これらは SDL-PUT-ENV を使用して設定できる。

パラメータ
  • WIDTH はウィンドウのピクセル幅であり、整数型（ INTEGER ）である。
  • HEIGHT はウィンドウのピクセルでの高さであり、整数型（ INTEGER ）である。もし WIDTH と HEIGHT が両方 0 である場合、カレントのビデオモード（あるいは、モードが設定されていない場合、デスクトップモード）の幅と高さが使用される。
  • BPP はピクセルあたりのビット数である。デフォルトは、カレント表示に於けるピクセルあたりのビット数を示す 0 である。注釈： 24 の BPP は、 3 バイト／ピクセルのパックド形式を使用する。より一般的な 4 バイト／ピクセルモードは、 32 の BPP を使用する。

  • ICON-CAPTION は、ウィンドウが最小化された時に現れるタイトルであり、文字列型（ STRING ）である。

［マクロ］ with-events (&optional type) declaration* statement* => result

マクロ WITH-EVENTS はメインのゲームループを管理する為の便利なマクロである。それは、やって来る SDL イベントを処理し、ゲームループを指定された秒あたりのフレーム数に制限する。

SDL-POLL-EVENT および SDL-WAIT-EVENT 両方のイベントの仕組みは、 TYPE をそれぞれ :POLL あるいは :WAIT として指定することにより対応されている。

注釈： マクロ WITH-EVENTS は、ビデオモード設定に使用されたスレッドと同じスレッドで呼び出される必要がある。

例
(SDL:WITH-EVENTS (:POLL)
  (:QUIT-EVENT () T)
  (:KEY-DOWN-EVENT (:KEY KEY)
      (WHEN (SDL:KEY= KEY :SDL-KEY-ESCAPE)
        (SDL:PUSH-QUIT-EVENT)))
  (:VIDEO-EXPOSE-EVENT () (SDL:UPDATE-DISPLAY))))))
フレームレートの制限

フレームレートはアクセサ FRAME-RATE を使用して指定される。フレームレートを毎秒 60 フレームに設定する例。： 

(SETF (SDL:FRAME-RATE) 60)
イベントの文法

イベントは、 (:EVENT-TYPE (&KEYS KEYS)) 形式で指定される。

  • EVENT-TYPE は、次のいずれかのキーワードである必要がある。： :ACTIVE-EVENT 、:KEY-DOWN-EVENT 、 :KEY-UP-EVENT 、 :MOUSE-MOTION-EVENT 、 :MOUSE-BUTTON-DOWN-EVENT 、 :MOUSE-BUTTON-UP-EVENT 、 :JOY-AXIS-MOTION-EVENT 、 :JOY-BUTTON-DOWN-EVENT 、 :JOY-BUTTON-UP-EVENT 、 :JOY-HAT-MOTION-EVENT 、 :JOY-BALL-MOTION-EVENT 、 :VIDEO-RESIZE-EVENT 、 :VIDEO-EXPOSE-EVENT 、 :SYS-WM-EVENT 、 :QUIT-EVENT 、 :USER-EVENT 、 :IDLE 。
  • KEYS は、返すイベントのメンバを指定し、各イベント種別固有である。これらの詳細は以下で述べる。

注釈： :QUIT-EVENT は、マクロ WITH-EVENT を終了する為に T を返す必要がある。

注釈： TYPE が :WAIT である時、 :IDLE は無視される。

イベントのポーリング

TYPE が :POLL である時、マクロ WITH-EVENTS は、現在保留となっているイベントを継続的にポーリングする。利用可能なイベントがない場合、ゲームループが走らされて、 :IDLE にあるフォームが実行される。

イベントを待つ

TYPE が :WAIT である時、マクロ WITH-EVENTS は、次に利用可能なイベントまで無期限にスリープする。利用可能なイベントがない場合、ゲームループは一時停止する。

:IDLE イベント
 (:IDLE ()
    &BODY BODY)

:IDLE イベントは、 SDL により生成されるものではないという点で特殊である。むしろ、 :IDLE のフォームは、イベントキューが空になった後、各ゲームループの度に実行される。 TYPE により指定されたイベントの仕組みが :WAIT である時、 :IDLE は無視される。

アクティブイベント
(:ACTIVE-EVENT (:GAIN GAIN :STATE STATE)
   &BODY BODY)

マウスがウィンドウ領域から出たり入ったりする時、 SDL-APP-MOUSE-FOCUS 型アクティベーションイベントが生成される。マウスがウィンドウに入った場合、 GAIN は 1 になり、それ以外の場合、 GAIN は 0 になる。アプリケーションがキーボードフォーカスを失ったり得たりする時（通常別のアプリケーションがアクティブになった時）、 SDL-APP-INPUT-FOCUS 型のアクティベーションイベントが発生する。最後に、アプリケーションが最小化／アイコン化される（ GAIN は 0 ）、あるいは元のサイズに戻された時、いずれも SDL-APP-ACTIVE 型のイベントが発生する。単一のイベントは STATE に複数の値を設定できる。注釈： このイベントは、アプリケーションウィンドウが最初に生成された時には発生しない。

  • GAIN は、イベントが損失の場合は 0 、利得の場合は 1 である。
  • STATE は次の値のビットマスクである。： マウスフォーカス取得喪失の場合、SDL-APP-MOUSE-FOCUS 。入力フォーカス取得喪失の場合、 SDL-APP-INPUT-FOCUS 。アプリケーションがアイコン化された場合、 SDL-APP-ACTIVE で GAIN は 0 。元のサイズに戻された場合の GAIN は 1 。
キーボードイベント
(:KEY-DOWN-EVENT (:STATE STATE :SCANCODE SCANCODE :KEY KEY :MOD MOD :UNICODE UNICODE)
   &BODY BODY)
(:KEY-UP-EVENT (:STATE STATE :SCANCODE SCANCODE :KEY KEY :MOD MOD :UNICODE UNICODE)
   &BODY BODY)

キーボードイベントは、一般にキーが放された時あるいはキーが押された時に発生する。イベントを生成したキーに関する情報は、 KEY および MOD に保存される。

キー :SDL-KEY-CAPSLOCK および :SDL-KEY-NUMLOCK は特殊な例であり、最初に押下された時に SDL-KEY-DOWN を報告し、放されて再度押下した時に SDL-RELEASED を報告する。したがって、これらのキーの KEYUP および KEYDOWN イベントは、キー自体ではなく Caps Lock および Num Lock の LED 状態に似ている。これらの特殊な例は、 Sun ワークステーションとの互換性の為に必要である。

注釈： SDL-ENABLE-KEY-REPEAT を使用してキーリピートが有効である場合、 SDL-KEY-DOWN イベントの繰り返しが発生する。

  • キーが押された場合、あるいは放された場合、 STATE はそれぞれ SDL-PRESSED あるいは SDL-RELEASED である。
  • SCANCODE は、キーボードにより返されるハードウェア依存のスキャンコードである。
  • KEY は、イベントが生成したキーの SDL 定義値である。キーの SDL 定義値は通常、次の形式をとる。： 数字キーの場合、 :SDL-KEY-0 から :SDL-KEY-9 。範囲 a-z のアルファベットキーの場合、 :SDL-KEY-a から :SDL-KEY-z 。その他は一般的に綴られており、例えば、 :SDL-KEY-PAGEDOWN 、 :SDL-KEY-F1 、 :SDL-KEY-NUMLOCK である。
  • MOD は、 SDL_GetModState で説明されているキーボード修飾子の現在の状態である。次の一つ以上。 :SDL-KEY-MOD-NONE 、 :SDL-KEY-MOD-LSHIFT 、 :SDL-KEY-MOD-RSHIFT 、 :SDL-KEY-MOD-LCTRL 、 :SDL-KEY-MOD-RCTRL 、 :SDL-KEY-MOD-LALT 、 :SDL-KEY-MOD-RALT 、 :SDL-KEY-MOD-LMETA 、 :SDL-KEY-MOD-RMETA 、 :SDL-KEY-MOD-NUM 、 :SDL-KEY-MOD-CAPS 、 :SDL-KEY-MOD-MODE 、 :SDL-KEY-MOD-RESERVED 。
  • UNICODE は変換された文字である。 unicode フィールドは、 UNICODE 変換が SDL_EnableUNICODE で有効である時にのみ使用される。 unicode が非 0 である場合、これはキー押下に対応する UNICODE 文字である。文字の上位 9 ビットが 0 である場合、これは同等の ASCII 文字に写像される。

※ SDL-KEY-DOWN とある箇所は本当か？　:SDL-KEY-DOWN はカーソルキーの「↓」を示す値である。

※キーを示すシンボルは、keysym.lisp で定義されている。

  • CODE は、ユーザ定義のイベントコードである。
  • DATA1 は、ユーザ定義のデータポインタである。
  • DATA2 は、ユーザ定義のデータポインタである。

［マクロ］ with-init flags declaration* statement* => result

マクロ WITH-INIT は、 BODY でフォームを実行する前に SDL ライブラリと SDL サブシステムの初期化を試みる便利なマクロである。マクロ WITH-INIT 終了の際、 SDL ライブラリと SDL サブシステムを初期化解除する。

lispbuilder-sdl 初期化ルーチンは、Lisp 開発環境が外部ライブラリを１回ロードする一方で、ライブラリを複数回初期化および初期化解除する事実により、やや複雑である。Ｃ／Ｃ＋＋開発環境は、オープンして各実行後にライブラリをクローズし、不完全あるいは不具合ある初期化解除関数によりハングされた全てのリソースを解放する。そのため、ライブラリが再初期化される前にリソースが適切に供給されない時、Ｃのライブラリは Lisp 環境に於いて頻繁にコアダンプする。

LISPBUILDER-SDL は、外部ライブラリの初期化／初期化解除をより細かく制御できるようにする機能を、プログラマに提供する。これらの機能を提供する関数は次のとおりである。：

  • 関数 INITIALIZE-ON-STARTUP
  • 関数 QUIT-ON-EXIT
  • 関数 LIST-SUB-SYSTEMS
  • 関数 RETURN-SUB-SYSTEMS-OF-STATUS
  • 関数 INIT-SUB-SYSTEMS
  • 関数 QUIT-SUB-SYSTEMS
  • 関数 INIT-SDL
  • 関数 QUIT-SDL
デフォルト
  • デフォルトでマクロ WITH-INIT は、 SDL サブシステムの SDL-INIT-VIDEO のみを初期化する。追加の SDL サブシステムは、関数 INITIALIZE-ON-STARTUP を呼び出すことにより初期化できる。
  • デフォルトで、マクロ WITH-INIT は、 SDL サブシステムの SDL-INIT-VIDEO のみを初期化解除する。追加の SDL サブシステムは、関数 QUIT-ON-EXIT を呼び出すことにより初期化解除できる。
SDL ライブラリの初期化／初期化解除

SDL ライブラリは次の場合のみ初期化される。：

  • ライブラリがまだ初期化されていない場合、あるいは
  • 関数 SDL-INIT-ON-STARTUP の結果が T である場合。

SDL ライブラリは次の場合のみ初期化解除される。：

  • 関数 SDL-QUIT-ON-EXIT の結果が T である時。
外部ライブラリの初期化／初期化解除

SDL ライブラリの初期化あるいは初期化解除に続いて外部ライブラリを自動的に初期化あるいは初期化解除できるようにするフックが提供されている。

外部ライブラリの初期化するには、外部ライブラリを初期化する関数を *EXTERNAL-INIT-ON-STARTUP* へプッシュする。関数は引数なしの必要がある。例： 

(defun init-ttf ()
   (if (is-init)
     t
     (sdl-ttf-cffi::ttf-init)))
(pushnew 'init-ttf sdl:*external-init-on-startup*)

外部ライブラリの初期化解除するには、外部ライブラリを初期化解除する関数を *EXTERNAL-QUIT-ON-EXIT* へプッシュする。関数は引数なしの必要がある。例： 

(defun quit-ttf ()
   (if (is-init)
     (sdl-ttf-cffi::ttf-quit)))
(pushnew 'quit-ttf sdl:*external-quit-on-exit*)
パラメータ
  • FLAGS は次のいずれか（複数可）にできる。：SDL-INIT-EVERYTHING 、 SDL-INIT-VIDEO 、 SDL-INIT-CDROM 、 SDL-INIT-AUDIO 、 SDL-INIT-TIMER 、 SDL-INIT-JOYSTICK 、 SDL-INIT-EVENTTHREAD 、 SDL-INIT-NOPARACHUTE 。注釈： マクロ WITH-INIT により FLAGS が設定される時、続く関数 INITIALIZE-ON-STARTUP および関数 QUIT-ON-EXIT への呼出は無視される。マクロ WITH-INIT は FLAGS で指定されたサブシステムのみ初期化／初期化解除する。
例
(with-init (SDL-INIT-VIDEO SDL-INIT-CDROM SDL-INIT-AUDIO)
  ....)

(with-init ()
  (INITIALIZE-ON-STARTUP SDL-INIT-VIDEO SDL-INIT-CDROM SDL-INIT-AUDIO)
  (QUIT-ON-EXIT SDL-INIT-VIDEO SDL-INIT-CDROM SDL-INIT-AUDIO)
  ....)

Draw a polygon of *DEFAULT-COLOR* to *DEFAULT-SURFACE*.

Local Methods

A vertex may be added using:

  • ADD-VERTEX which accepts an SDL:POINT, or
  • ADD-VERTEX-* which is the x/y spread version

ADD-VERTEX and ADD-VERTEX-* are valid only within the scop of WITH-SHAPE.
